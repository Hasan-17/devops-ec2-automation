# EC2 Automation Assignment 2: S3 Logging & Infrastructure

## Overview

This project automates the setup of an S3 bucket for log archival and configures AWS resources using shell scripts and Terraform. The workflow includes provisioning infrastructure, uploading logs, and verifying the S3 bucket contents.

---

## Prerequisites

- **AWS Account** with sufficient permissions.
- **AWS CLI** configured with credentials on your machine.
- **Terraform** installed ([Download](https://www.terraform.io/downloads.html))
- **Bash** (Linux/macOS or WSL on Windows)

---

## Project Structure

```
scripts/             # Shell scripts for automation
  setup_infra.sh
  upload_logs.sh
  verify_s3_list.sh
config/              # Environment configuration files
  env_config.env
  secret.env
terraform/           # Terraform configuration files
  main.tf
  outputs.tf
  provider.tf
  variables.tf
  terraform.tfstate          # Auto-generated by Terraform
  terraform.tfstate.backup   # Auto-generated by Terraform
```

---

## Setup Instructions

1. **Clone the Repository**
   ```sh
   git clone https://github.com/Hasan-17/devops-ec2-automation.git
   cd devops-ec2-automation
   ```

2. **Configure AWS Credentials**
   -  environment variables as required in `config/secret.env`.

3. **Review/Edit Configuration Files**
   - Update `config/env_config.env` with your parameters (e.g., region, S3 bucket name).

---

## Usage

### 1. Provision Infrastructure with Terraform

```sh
cd terraform
terraform init
terraform apply
```
- Review the `main.tf`, `variables.tf` etc. for resource details.
- Confirm resource creation when prompted.

### 2. Run Log Upload Script

```sh
cd ../scripts
./upload_logs.sh
```
- This uploads local logs to the configured S3 bucket.

### 3. Verify S3 Bucket Contents

```sh
./verify_s3_list.sh
```
- Checks and lists the logs present in your S3 bucket.

---

## Cleanup

To destroy all provisioned resources:

```sh
cd ../terraform
terraform destroy
```

---

## Troubleshooting

- If AWS CLI commands fail, check your credentials and permissions.
- Ensure all environment variables in `env_config.env` are properly set.
- For any Terraform errors, run `terraform plan` to preview and debug.

---

## Notes

- All scripts are bash shell scripts; ensure they are executable (`chmod +x scriptname.sh`).
- Do **not** commit your AWS secrets or sensitive information.
- `.gitignore` should be configured to avoid pushing sensitive files.
